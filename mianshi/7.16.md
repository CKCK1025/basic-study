1. 关于redux与flux区别
flux的更新逻辑放在store,且包含多个store。
redux中没有dispatcher的概念
redux利用所有action都会经过Store.dispatch()方法，在dispatch action后，到达store之前，插入中间件，进行操作。promise,async,thunk,saga等



2. redux中如何进行异步操作?
异步中间件进行处理，saga,thunk等；




3. redux中间件实现原理，关于面向切面编程。
import { createStore, applyMiddleware } from 'redux'
// 创建 store
const store = createStore(
    reducer,
    initial_state,
    applyMiddleware(middleware1, middleware2, ...)  //引入中间件
);


4. Hoc和mixin
实现一个函数，传入一个组件，然后在函数内部再实现一个函数去扩展传入的组件，最后返回一个新的组件，
这就是高阶函数的概念，作用就是为了更好的扩展。
mix隐含了一些依赖，多个mixin可能存在相同的函数名，多个组件在使用mixin,导致后面不好修改或添加
mixin中的方法。


5. 事件机制
阻止冒泡用event.preventDefault
实现合成事件的目的：
合成事件首先磨平了浏览器兼容问题，原生事件的包装器，扩展开发能力。
原生浏览器事件会创建一个事件对象，太多事件对象内存分配问题。合成事件来说，专门有一个事件来管理创建和销毁。
